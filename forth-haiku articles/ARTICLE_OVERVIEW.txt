FORTH HAIKU ARTICLE SERIES - STRUCTURE
=======================================

A 6-part series exploring compiler design and software architecture
through the lens of haiku.js

OVERVIEW
--------

Part 1: Overview - Introduction to two-phase compilation
Part 2: Examples - Concrete compilation walkthroughs  
Part 3: Pipeline Architecture - Unix pipelines as system design
Part 4: Optimizer - Deep dive into Phase 2 algorithms
Part 5: Dictionary - Deep dive into Phase 1 code generation
Part 6: Conclusion - Lessons and principles

NARRATIVE ARC
-------------

Parts 1-2: WHAT (What the compiler does)
  → Introduces the two-phase approach
  → Shows concrete examples of transformation

Part 3: WHERE (Where these principles apply)
  → Extends to system architecture
  → Demonstrates Unix pipeline philosophy

Parts 4-5: HOW (How it works internally)
  → Phase 2 optimizer algorithms
  → Phase 1 dictionary generation

Part 6: WHY (Why this matters)
  → Broader lessons about software design
  → Practical applications of principles

KEY THEMES
----------

• Separation of Concerns - Each part does one thing
• Composability - Simple pieces create complexity
• Clear Interfaces - Well-defined boundaries
• Simplicity - Achieved through composition
• Practical Engineering - Working techniques, not theory
